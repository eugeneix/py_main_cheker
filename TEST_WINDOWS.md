# Тестирование на Windows

## Быстрый запуск для теста

### Вариант 1: Через командную строку (CMD)

```cmd
set HEADLESS=false
python web_monitor.py "https://www.rtoperator.ru/" "ВАШ_ТОКЕН" "ВАШ_CHAT_ID" "auto" "Рождество 2026"
```

### Вариант 2: Через PowerShell

```powershell
$env:HEADLESS = "false"
python web_monitor.py "https://www.rtoperator.ru/" "ВАШ_ТОКЕН" "ВАШ_CHAT_ID" "auto" "Рождество 2026"
```

### Вариант 3: Использовать готовый скрипт

1. Отредактируйте `test_windows.bat` или `test_windows.ps1`
2. Замените `ВАШ_ТОКЕН` и `ВАШ_CHAT_ID` на реальные значения
3. Запустите скрипт

## Что изменилось для Windows:

1. ✅ **Вывод в консоль**: Теперь при каждой проверке выводится сообщение:
   - `✅ Элемент на месте!` - если элемент найден
   - `⚠️ Элемент отсутствует!` - если элемент не найден
   - `⚠️ Элемент изменился!` - если текст изменился

2. ✅ **Headless режим**: Можно отключить через переменную окружения `HEADLESS=false`
   - С `HEADLESS=false` браузер будет виден (удобно для теста)
   - С `HEADLESS=true` или без переменной - браузер скрыт (для production)

3. ✅ **User-Agent**: Автоматически определяется в зависимости от ОС

## Установка зависимостей на Windows:

```powershell
# Установите Python 3.10+ если еще не установлен
# Скачайте с python.org

# Установите зависимости
pip install -r requirements.txt

# Убедитесь, что Chrome установлен
# ChromeDriver будет автоматически загружен Selenium
```

## Остановка теста:

Нажмите `Ctrl+C` в консоли

## Примечания:

- Для теста можно использовать фиктивный токен (приложение проверит подключение и выдаст ошибку, но можно увидеть работу)
- Или создайте реального бота через @BotFather для полного теста
- Если элемент не найден, вы увидите `⚠️ Элемент отсутствует!` каждую минуту
